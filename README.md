# Нейро-консультант на FastAPI (RAG)

Чат-консультант по базе знаний, развёрнутый как полноценный REST API-сервис, а не скрипт. Отвечает на вопросы пользователя строго по загруженному документу, используя retrieval-augmented generation (RAG).

## Что умеет
- Принимает вопрос по HTTP и возвращает ответ на основе базы знаний
- Ищет релевантные фрагменты документа через векторный поиск (FAISS)
- Считает количество обращений к сервису
- Не отвечает на вопросы вне темы документа (защита от галлюцинаций)

## Стек
- **Python**, **FastAPI** — REST API
- **LangChain**, **OpenAI API** (GPT-3.5/4) — работа с LLM
- **FAISS** + **OpenAIEmbeddings** — векторная база и семантический поиск
- **pydantic** — валидация входных данных
- **python-dotenv** — хранение ключей в `.env`

## Архитектура
RAG-логика вынесена в отдельный переиспользуемый класс (`chunks.py`), слой API — отдельно (`main.py`). Это разделение ответственности: бизнес-логику можно использовать и без веб-сервиса.

## Запуск
```bash
pip install -r requirements.txt
# создать файл .env и вписать туда OPENAI_API_KEY=ваш_ключ
uvicorn main:app --reload --port 5000
```

## Эндпоинты
- `POST /api/get_answer` — задать вопрос, получить ответ
- `GET /api/requests_count` — число обращений к сервису

## Чему проект научил
Упаковывать RAG-решение в рабочий сервис с API, разделять логику и слой представления, применять production-практики (переменные окружения, фиксация зависимостей, обработка ошибок).
