# Нейро-консультант на FastAPI (RAG)

Чат-консультант по базе знаний, развёрнутый как полноценный REST API-сервис, а не скрипт. Отвечает на вопросы пользователя строго по загруженному документу, используя retrieval-augmented generation (RAG).

## Что умеет
- Принимает вопрос по HTTP и возвращает ответ на основе базы знаний
- Ищет релевантные фрагменты документа через векторный поиск (FAISS)
- Считает количество обращений к сервису
- Не отвечает на вопросы вне темы документа (защита от галлюцинаций)

## Стек
- **Python**, **FastAPI** — REST API
- **LangChain**, **OpenAI API** (GPT-3.5/4) — работа с LLM
- **FAISS** + **OpenAIEmbeddings** — векторная база и семантический поиск
- **pydantic** — валидация входных данных
- **python-dotenv** — хранение ключей в `.env`

## Архитектура
RAG-логика вынесена в отдельный переиспользуемый класс (`chunks.py`), слой API — отдельно (`main.py`). Это разделение ответственности: бизнес-логику можно использовать и без веб-сервиса.

## Запуск
```bash
pip install -r requirements.txt
# создать файл .env и вписать туда OPENAI_API_KEY=ваш_ключ
uvicorn main:app --reload --port 5000
```

## Эндпоинты
- `POST /api/get_answer` — задать вопрос, получить ответ
- `GET /api/requests_count` — число обращений к сервису

## Чему проект научил
Упаковывать RAG-решение в рабочий сервис с API, разделять логику и слой представления, применять production-практики (переменные окружения, фиксация зависимостей, обработка ошибок).

*Учебный проект программы «ChatGPT Professional», Университет искусственного интеллекта.*

# Telegram-бот на Python (async)

Telegram-бот с асинхронной обработкой сообщений: реагирует на текст, фото и голос, поддерживает команды, интерактивные кнопки и переключение языка интерфейса.

## Что умеет
- Обрабатывает текстовые, голосовые сообщения и фотографии
- Команды `/start`, `/help` и интерактивные inline- и reply-клавиатуры
- Мультиязычный интерфейс (русский / английский) с сохранением выбора пользователя
- Сохраняет присланные медиафайлы на диск с уникальными именами

## Стек
- **Python**, **python-telegram-bot**
- Асинхронное программирование (**async/await**, **asyncio**)
- **python-dotenv** — хранение токена в `.env`

## Запуск
```bash
pip install python-telegram-bot python-dotenv
# создать .env и вписать TG_TOKEN=токен_от_BotFather
python simple_bot.py
```

## Структура
- `simple_bot.py` — основной бот с выбором языка и обработкой медиа
- `handlers.py` — пример обработки команд и типов сообщений
- `buttons.py` — inline- и reply-клавиатуры
- `media_files.py` — приём и сохранение фото/голоса

## Чему проект научил
Строить асинхронных Telegram-ботов, обрабатывать разные типы контента, делать интерактивный интерфейс с клавиатурами и состоянием пользователя. Выполнено на Pro-уровне — без шаблона занятия.

## 3. Репозиторий: `neuro-copywriter-video`

==================== копируй отсюда ====================

# Нейро-копирайтер по видео (мультимодальный RAG)

Пайплайн, который превращает видео, аудио и текстовые документы в единую базу знаний, а затем строит по ней нейро-консультанта и автоматически генерирует обучающую методичку.

## Как работает
1. Извлекает аудиодорожку из видео YouTube (`yt-dlp`) и скачивает файлы с Google Drive (`gdown`)
2. Транскрибирует речь в текст моделью **Whisper**
3. Объединяет тексты из видео, аудио и документа в один источник
4. С помощью GPT размечает «сырой» текст на логические разделы (Markdown)
5. Сжимает материал в методичку («убрать воду, оставить суть»)
6. Строит RAG-консультанта для ответов на вопросы по собранной базе

## Стек
- **Python**, **LangChain**, **OpenAI API**
- **Whisper** — распознавание речи
- **yt-dlp**, **gdown** — сбор данных из разных источников
- **FAISS** + **OpenAIEmbeddings** — векторная база
- Retry-логика для устойчивости к сбоям API

## Чему проект научил
Собирать сквозной пайплайн из источников разного типа (видео + аудио + текст), комбинировать распознавание речи и RAG, автоматизировать генерацию контента и закладывать устойчивость к сбоям.

*Учебный проект программы «ChatGPT Professional», Университет искусственного интеллекта.*

# Транскрибация звонков и сравнение STT-моделей

Перевод аудиозаписи телефонного разговора в текст с автоматическим разделением реплик участников, плюс сравнение разных моделей распознавания речи по скорости и качеству.

## Что внутри
- Обработка аудио через **PyDub** (нарезка, конвертация частоты/разрядности, работа с каналами)
- Разделение стереозаписи на моноканалы — раздельная транскрибация менеджера и клиента
- Два подхода к распознаванию: облачный **Tinkoff VoiceKit** и локальный **Whisper**
- Бенчмарк всех размеров Whisper (`tiny` → `large`) с замером времени и графиком сравнения

## Стек
- **Python**, **Whisper**, **Tinkoff VoiceKit**, **SpeechRecognition**
- **PyDub** (`ffmpeg`) — обработка аудио
- **matplotlib** — визуализация сравнения моделей

## Вывод проекта
Сравнение показало баланс между скоростью и качеством распознавания у разных размеров модели — это помогает осознанно выбирать модель под конкретную задачу, а не брать наугад.

## Чему проект научил
Работать с аудиосигналом, применять облачные и локальные STT-сервисы, разделять речь по участникам диалога и обоснованно выбирать инструмент через бенчмарк.

*Учебный проект программы «ChatGPT Professional», Университет искусственного интеллекта.*
